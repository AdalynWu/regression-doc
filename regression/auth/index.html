身分驗證與進站邏輯 (Auth & Routing) ​

QA 團隊與新人 Onboard 知識點

模組業務目的： 確保所有流量（台灣、海外）能順利進入平台，並驗證所有登入/註冊機制（包含傳統與第三方驗證）的暢通。此模組為全站最重要的 P0 防守區，任何阻斷都會直接影響營收。

環境與注意事項：

  1. 海外測試： 需準備穩定可用的 VPN，切換至非台灣地區 IP（建議美國或日本）。
  2. 全新狀態： 測試導流邏輯前，請務必開啟「無痕視窗」或清除瀏覽器 Cache/Cookie。
  3. 驗證碼機制： 若無特殊工程測試 API，需實際接收 Email 或簡訊驗證碼。
  4. 防機器人驗證： 所有原生註冊/登入行為，皆須通過「滑塊拼圖 (Geetest)」驗證。

一、地區導流與進站邏輯 (Routing) ​

[ROUT-01] 台灣地區全新使用者進站 (Taiwan Organic) ​

狀態測試步驟 (業務邏輯)預期結果
1. 於無紀錄狀態，確認 IP 為台灣並輸入網站首頁網址成功載入 Landing Page (品牌宣傳頁)，未直接進入內容大廳。
2. 點擊右上角「帳號註冊/登入」順利彈出標準的登入/註冊彈窗。

[ROUT-02] 海外地區全新使用者進站 (Overseas Organic) ​

狀態測試步驟 (業務邏輯)預期結果
1. 於無紀錄狀態，使用 VPN 將 IP 切換至海外地區網路環境設定完成。
2. 輸入網站首頁網址1. 略過 Landing Page。
2. 畫面直接導向主站大廳 (Explore Tab)。
3. 頁面處於未登入的訪客狀態。

[ROUT-03] 登出後再次進站 (Returning User) ​

狀態測試步驟 (業務邏輯)預期結果
1. 正常登入後，點擊「登出」並重新整理首頁網址畫面正常載入 Landing Page。
2. 點擊右上角「帳號註冊/登入」彈出「歡迎回來」視窗，並預設帶出上次使用的帳號與登入按鈕。
3. 點擊「使用其他方式」可順利退回一般的登入選擇畫面。

二、常規登入流程 (Login) ​

[LOGIN-01] 手機號碼與 Email 登入 (含滑塊驗證) ​

狀態測試步驟 (業務邏輯)預期結果
1. 於登入視窗選擇對應方式，輸入「已註冊」的手機或 Email欄位無報錯。
2. 點擊「取得驗證碼」並拖曳滑塊完成拼圖1. 顯示「驗證通過」綠色提示。
2. 畫面跳轉至「輸入驗證碼」頁面，並開始倒數計時。
3. 測試國碼切換 (僅限手機登入)：點擊手機輸入框前方的 +886彈出國家列表，可支援搜尋 (如輸入 Taiwan)，並成功切換國碼。
4. 輸入正確的 6 碼驗證碼並送出登入成功，畫面導向主站大廳。

[LOGIN-02] 帳號密碼登入 ​

狀態測試步驟 (業務邏輯)預期結果
1. 點擊「使用帳號密碼登入」，輸入正確的帳號 (用戶名稱) 與密碼欄位切換正確，密碼可正常輸入。
2. 完成滑塊驗證並點擊「登入」登入成功，畫面導向主站大廳。

[LOGIN-03] 第三方登入與多語系 (SSO & Localization) ​

狀態測試步驟 (業務邏輯)預期結果
1. 點擊 Google 或 Apple 登入按鈕成功呼叫對應的第三方授權視窗，授權後能順利登入。
2. 點擊登入視窗下方的語系按鈕 (如「繁體中文」) 並切換為 English整個登入彈窗的文案 (包含 placeholder) 即時切換為對應語言。

[LOGIN-04] 訪客一鍵登入與生命週期 (Guest Login) ​

測試要點

訪客帳號為「一次性 Session」。一旦登出，該訪客帳號即刻失效，所有未綁定的資產與瀏覽紀錄將無法找回。

狀態測試步驟 (業務邏輯)預期結果
1. 於 Landing Page 點擊「遊客進站」 (粉紅色按鈕)略過所有登入/註冊驗證，畫面直接導向主站大廳 (Explore Tab)。
2. 點擊右上角頭像檢查身分狀態1. 成功獲取一組帶有 guest_ 開頭的隨機亂碼帳號。
2. 頭像下方顯示「立即綁定帳號保留鑽石」的黃色引導區塊。
3. 測試生命週期：於設定中點擊「登出」並確認成功登出並導回 Landing Page。
4. 再次點擊「遊客進站」系統配發一組全新的 guest_ 隨機帳號，舊有訪客資料已被清除。

三、註冊與密碼救援 (Signup & Recovery) ​

[SIGN-01] 全新帳號註冊流程 (含年齡驗證與邀請碼) ​

狀態測試步驟 (業務邏輯)預期結果
1. 於登入視窗輸入「未註冊」的手機/Email，並完成滑塊與驗證碼系統判定為新用戶，導向「建立帳號 / 帳號資訊」頁面。
2. 輸入符合規則的用戶名稱 (3-20 字元)欄位狀態正常，無錯誤提示。
3. 點擊生日欄位，選擇一個大於 18 歲的日期系統接受該日期。
4. (選測) 輸入有效的邀請碼，點擊「完成註冊」1. 成功註冊並導向主站。
2. 檢查資產，應發放對應的新手獎勵 (如：5,000 鑽石)。

[RECOV-01] 忘記密碼流程 ​

狀態測試步驟 (業務邏輯)預期結果
1. 於帳密登入模式，點擊「忘記密碼？」導向「忘記密碼」驗證頁面。
2. 輸入已綁定此帳號的手機/Email，完成滑塊驗證導向「輸入驗證碼」頁面，並開始 60 秒倒數。
3. 輸入正確驗證碼並送出導向「設定新密碼」頁面，重設完成後能以新密碼成功登入。

四、進階登入機制 (Passkey) ​

[PASS-01] 生物辨識開通與免密碼登入 ​

狀態測試步驟 (業務邏輯)預期結果
1. 未綁定用戶：一般登入進入首頁系統彈出推廣彈窗：「立即開通臉部指紋登入快速進站」。
2. 點擊「立即開通」並完成系統生物辨識 (Face ID/Touch ID) 綁定成功將當前設備綁定至該帳號。
3. 已綁定用戶：登出後再次進站頁面底部顯示簡化的歡迎彈窗，點擊即可直接呼叫生物辨識進行秒登入。

五、特殊流量進站與訪客權限 (AD Traffic & Guest Rules) ​

[ROUT-04] 廣告連結進站邏輯 (AD Traffic) ​

狀態測試步驟 (業務邏輯)預期結果
1. 於無紀錄狀態下，使用帶有特定廣告參數的網址進站 (例如：?ad_source=xxx)略過標準 Landing Page，畫面直接導向該廣告所設定的目標頁面（如特定直播間或創作者個人頁）。
2. 檢查當前身分狀態系統自動配發訪客 (Guest) 身分，以便用戶立即觀看內容。

[GUEST-01] 訪客點擊付費功能之攔截 (Premium Feature Wall) ​

狀態測試步驟 (業務邏輯)預期結果
1. 以訪客身分登入，點擊需要消耗鑽石的付費功能（例如：解鎖私密影片、發送付費禮物）系統中斷操作，並彈出「付費轉換/首購限定優惠」的引導視窗。
2. 點擊視窗上的「立即解鎖」或註冊按鈕畫面順利導向註冊或儲值流程，完成後需能接續原本的付費動作。

[GUEST-02] 訪客點擊一般互動功能之攔截 (Social Feature Wall) ​

狀態測試步驟 (業務邏輯)預期結果
1. 以訪客身分登入，點擊一般社交互動功能（例如：點擊追蹤、按讚）系統中斷操作，並彈出一般的「歡迎來到 SWAG / 註冊引導」視窗。
2. 點擊「註冊/登入」按鈕成功喚起標準登入視窗。

六、異常狀態與風險控管 (Risk Control) ​

[BLOCK-01] 違規停權帳號攔截 (Banned Account Hard Block) ​

注意

同一 fingerprint 短時間內重複註冊訪客帳號會造成 user banned

狀態測試步驟 (業務邏輯)預期結果
1. 準備一組在後台已被標記為「停權/Banned」的測試帳號確認該帳號狀態為異常。
2. 於登入視窗輸入該帳號與密碼，並完成滑塊驗證送出1. 登入成功但無法進入主站。
2. 畫面強制彈出「帳號已暫時限制」提示視窗。
3. 嘗試關閉視窗或點擊背景空白處該視窗沒有關閉 (X) 按鈕，點擊背景亦無法關閉，用戶遭到強制阻斷 (Hard Block)。
4. 點擊視窗內的「聯絡客服」按鈕成功喚起線上客服系統或跳轉至客服信箱。
