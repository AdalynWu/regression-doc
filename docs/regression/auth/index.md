# 身分驗證與進站邏輯 (Auth & Routing)

::: info QA 團隊與新人 Onboard 知識點
**模組業務目的：**
確保所有流量（台灣、海外）能順利進入平台，並驗證所有登入/註冊機制（包含傳統與第三方驗證）的暢通。此模組為全站最重要的 P0 防守區，任何阻斷都會直接影響營收。

**環境與注意事項：**
1. **海外測試：** 需準備穩定可用的 VPN，切換至非台灣地區 IP（建議美國或日本）。
2. **全新狀態：** 測試導流邏輯前，請務必開啟「無痕視窗」或清除瀏覽器 Cache/Cookie。
3. **驗證碼機制：** 若無特殊工程測試 API，需實際接收 Email 或簡訊驗證碼。
4. **防機器人驗證：** 所有原生註冊/登入行為，皆須通過「滑塊拼圖 (Geetest)」驗證。
:::

:::

| <input type="checkbox"> | 3. 測試生命週期：於設定中點擊「登出」並確認 | 成功登出並導回 Landing Page。 |
| <input type="checkbox"> | 4. 再次點擊「遊客進站」 | 系統配發一組**全新**的 `guest_` 隨機帳號，舊有訪客資料已被清除。 |
