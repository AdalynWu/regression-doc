# 模組：訪客與會員登入 (Auth)

::: info 💡 給新人的 Onboard 知識點
**業務邏輯說明：**
我們網站的登入分為「訪客 (Guest)」與「正式會員」。訪客只能瀏覽內容，不能結帳；正式會員可以結帳。每週回歸測試時，必須確保這兩條路徑暢通，否則使用者無法進入主要漏斗。
**測試帳號準備：**
* 正式會員： `test_user@company.com` / `Aa123456`
* 訪客：點擊登入頁下方的 Guest 即可。
:::
