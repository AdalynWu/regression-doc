測試環境與帳號準備 (Environment & Accounts) ​

Onboard 知識點

本文件涵蓋前端測試環境的存取方式、測試帳號的建立規則，以及執行回歸測試時需準備的軟硬體設備與自動化工具。

一、前端 Web 測試環境 (Test Environments) ​

QA 團隊在進行前端每週版更或功能驗證時，主要透過 GitHub Pull Request (PR) 產生的專屬測試網址進行。

1. 取得測試網址 ​

每當前端發起 PR 並完成部署後，GitHub Actions 會自動生成對應版本的測試環境。

  • 網址格式： https://{version}.app.swag.live (例如：https://v3-277.app.swag.live)。
  • 快速存取： 你可以直接在 GitHub PR 頁面下方，點擊 View deployment 按鈕直接前往該測試站。

2. 後端資料串接注意事項 ​

重要警告

目前前端 Web 測試環境 串接的後端資料為正式環境 (Production)。 所有在測試站上的寫入行為（包含發文、私訊、修改資料）皆會同步至正式資料庫。請務必謹慎操作，避免影響真實使用者。

二、測試帳號申請規則 (Test Accounts) ​

為了覆蓋不同的權限漏斗與金流測試，QA 成員必須自備多組測試帳號。

1. 信箱分身技巧 (Alias) ​

請使用公司配發的 Email 進行申請。利用 Gmail 的加號 + 標籤功能，可以使用同一個信箱註冊無數個測試帳號。

  • 範例：
    • {your.name}+1@swag.live
    • {your.name}+2@swag.live
    • {your.name}+3@swag.live

2. 必備的測試帳號層級 ​

每位 QA 至少需準備以下四種層級的帳號，以利完整覆蓋回歸測試：

  1. 一般用戶 (Regular User)： 全新的白紙帳號，用於測試最標準的訪客與付費轉換流程。
  2. Beta 用戶 (Beta User)： 綁定 Beta Tag，用於測試 Live 環境下尚未公開的第三方金流商，或是部分為開放給一般用戶的 features。
  3. 主播 Beta 帳號 (Creator Beta)： 模擬創作者身分，此類帳號系統設定為「無法消耗鑽石」。
  4. 最高權限用戶 (God Mode/Admin)： 具備所有解鎖權限，可免費觀看所有付費內容，用於快速驗證內容播放狀態。

三、測試設備與輔助工具 (Devices & Tools) ​

1. 硬體設備規範 ​

  • iOS 裝置： 無特定機型限制，但每週回歸測試至少需使用 1 台 iOS 實體機進行驗證。
  • MDM 企業管理： 測試機器的安裝與憑證管理流程 (SOP 待補)。

2. 輔助測試工具 ​

  • Moor (API 測試)： 需使用安裝於測試機上的專屬測試 App 進行後端與 API 的交接驗證。
  • 自動化測試 (Automation)： 團隊目前採用 Robot Framework 執行自動化測試。相關環境建置請參閱自動化專案文件。
